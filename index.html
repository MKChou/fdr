系統簡介與研究動機

隨著高齡化社會的來臨，老年人跌倒造成的受傷風險日益提高。跌倒是老人住院與傷亡的主要原因之一，及早偵測並通知救援可顯著降低後續併發症與死亡率。

傳統跌倒偵測方法多依賴單一傳感器（可穿戴或視覺），但各有局限。文獻指出，僅用視覺於夜間或範圍外易產生漏偵，而加入音訊資訊可補強偵測能力。

核心特色

  • 多模態融合策略 - 結合視覺與音訊提升準確度
  • 邊緣運算平台 - 使用 Jetson Nano 高效低功耗運算
  • 隱私保護 - 本地處理，無需上傳雲端
  • 即時監控 - 符合實際應用需求

使用技術概述

MoveNet 姿勢估測

Google 提出的超快速、高精度人體姿勢檢測模型，可偵測人體 17 個關鍵點，達到 30 FPS 以上的即時速度。

Lightning 版本 Thunder 版本

卷積神經網路 (CNN)

深度學習模型，以捲積層學習影像的空間特徵結構，在物件與動作識別上表現卓越。

自動特徵學習 姿勢分類

梅爾頻率倒譜係數 (MFCC)

音訊處理領域常用的特徵，貼近人耳聽覺感知，能有效捕捉撞擊聲、摔倒聲等異常音訊特性。

Mel 頻率尺度 離散餘弦轉換

ONNX Runtime

跨平台執行預訓練模型，支援 CUDA GPU 加速與 CPU 回退，提供硬體加速與平行化運算。

GPU 加速 多線程支援

影像擷取與分析模組

使用攝影機持續擷取影像，偵測到人形後交給 MoveNet 進行關鍵點檢測，再將骨架資訊送入 CNN 模型進行姿勢分類。

音訊擷取與分析模組

開啟麥克風進行環境聲錄製，以固定間隔擷取音訊片段，計算 MFCC 特徵並輸入音訊分類模型。

融合判斷模組

收集影像與音訊模組的判斷分數，按設計的權重或融合規則進行整合，超過門檻時判定為跌倒事件。

多模態融合策略與演算法設計

1

資料擷取

同步取得當前視頻畫面與音訊片段

2

視覺分析

MoveNet 擷取骨架關鍵點，CNN 分析姿勢，輸出視覺分數 (0~1)

3

音訊分析

計算 MFCC 特徵，透過 ONNX 音訊模型輸出音訊分數

4

分數融合

按權重融合視覺與音訊分數，超過門檻值 (0.7) 則觸發警報

5

結果處理

跌倒判定時保存事件影像和時間戳，否則繼續下一輪偵測

實作平台說明與優化技巧

NVIDIA Jetson Nano 2GB

  • 尺寸：70×45mm 小型嵌入式模組
  • GPU：128核 Maxwell GPU
  • CPU：雙核心 ARM CPU
  • 記憶體：2GB RAM（資源有限）
  • 特色：低功耗邊緣運算，適合 AI 推論

關鍵優化技巧

CUDA 相容性

使用 ONNX Runtime 1.6 配合 CUDA 10.2，確保 GPU 加速功能正常運作，較新版本需 CUDA 11 支援。

記憶體管理

模型壓縮與量化（FP16/INT8），使用 I/O Binding 技術降低資料複製開銷，避免記憶體不足。

多線程架構

並行執行視訊擷取、音訊擷取與推論工作，充分發揮多核與 GPU 並行能力。

系統特色與優勢

多模態融合優勢

視覺與音訊互補，顯著提升跌倒偵測效果。系統敏感度從 81.7% 提升至 96.7%，真實場景偵測更可靠。

可改進與未來發展建議

增強模型能力

整合時間序列模型（3D-CNN、LSTM）捕捉跌倒動作的時間演變，加入注意力機制提升微小動作辨識。

整合至智能照護平台，自動發送警報到行動裝置，利用長時間監測數據進行使用者行為建模。

結語與心得

本專題實現了一套結合視覺與音訊的多模態跌倒偵測系統，在 Jetson Nano 2GB 平台上完成了端側即時推論。系統成功利用 MoveNet 與 CNN 分析人體姿勢，再結合 MFCC 特徵與音訊模型捕捉異常聲響，經分數融合後提高了偵測準確度。

在開發過程中，我們深入學習了 ONNX Runtime 在邊緣設備上的部署與優化技巧，也體會到資源有限的 Jetson Nano 在模型選擇與記憶體管理上的挑戰。未來若能持續改進模型與融合策略，並應用在更真實的場景中，本系統有望成為高齡者或病患照護中的重要輔助工具。
